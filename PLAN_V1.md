## 1. 计划目标

## 3. 验收关口（Gate）

### Gate-A（M2 后）

- [ ] 模板不再依赖硬编码常量
- [ ] 任务状态与阻塞链路可追溯

### Gate-B（M3 后）

- [ ] 阶段成果可一键导出
- [ ] 文件检索在真实项目可用

### Gate-C（M5 后，V1 关闭）

- [ ] 设计人员可快速定位当前阶段缺口
- [ ] 负责人可查看阶段进度与风险摘要
- [ ] 全流程无自动规范判断能力（保持 V1 边界）

## 4. 风险与应对

1. 需求膨胀风险
- 应对：严格按 Must/Should 执行，超出范围进入 V2 backlog

2. 数据兼容风险
- 应对：新增字段兼容老数据，避免一次性破坏式迁移

3. 交付节奏风险
- 应对：每个里程碑必须有可演示交付物，不以“开发中”作为完成标准
