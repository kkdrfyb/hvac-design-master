## 用户问题（第 1 轮）
结构视图目前设计输入、收到提资、计算结果等是竖向列表，展开的结果也是竖向列表，可以看到结果，但是展示效果不好，可以探讨下更优秀的展示方案，施工阶段可以做成二级标签。先不修改，给我几个方案选择，有示意图最好。

## Codex 最终答复（第 1 轮）
当前这个结构视图的问题很明确：

1. 一级分类和展开结果都沿纵向堆叠，层级一多就会变成长页面。
2. 用户能看到内容，但很难快速形成“这个子项整体完成到哪了”的空间感。
3. `施工阶段` 和前面几个阶段的展示逻辑其实不完全一样，现在被同一种纵向树强行套住了。

我建议你先在下面 3 个方案里选，不要继续在当前纵向树上小修。

**方案 1**
阶段页签 + 分类卡片墙 + 右侧详情抽屉

适合你现在这个项目，信息清楚，改造成本也相对可控。我认为这是当前最优先方案。

展示方式：
- 顶部先切阶段：`方案 / 初设 / 施工图`
- 中间不是竖向列表，而是 `设计输入 / 收到提资 / 计算结果 / 对外提资 / 输出成果 / 出图后闭环 / 设计备注` 这些分类卡片墙
- 每张卡片只显示：状态、完成度、文件数、最后更新时间、是否有风险
- 点击卡片后，在右侧抽屉里展示该分类下的细项、最终版文件、摘要

施工阶段：
- `施工图阶段` 单独加二级标签
- 建议二级标签先做成：`图纸成果`、`设计变更`、`问题闭环`、`归档版本`

示意图：
```mermaid
flowchart LR
    A["阶段标签\n方案 | 初设 | 施工图"] --> B["分类卡片墙"]
    B --> C1["设计输入"]
    B --> C2["收到提资"]
    B --> C3["计算结果"]
    B --> C4["对外提资"]
    B --> C5["输出成果"]
    B --> C6["出图后闭环"]
    C5 --> D["右侧详情抽屉\n细项 | 最终版文件 | 摘要"]
```

优点：
- 一眼能看到整体状态
- 不会被长列表拖垮
- 子项主页和设计过程页职责更清楚
- 很适合后续继续加颜色、风险、最终版标记

缺点：
- 细项很多时，还是需要抽屉或二级面板承接

**方案 2**
横向泳道板

这个更偏“管理看板”，适合快速比较各分类在同一阶段的状态。

展示方式：
- 一列一个大类
- 每列下面是细项卡片
- 卡片上显示文件数、最终版、状态、摘要
- 进行中、未开始、已完成通过颜色直接区分

施工阶段：
- `施工图阶段` 内再切二级标签
- 每个二级标签对应一组泳道

示意图：
```mermaid
flowchart TB
    A["施工图阶段"] --> B["二级标签: 图纸成果"]
    B --> C1["设计输入泳道"]
    B --> C2["收到提资泳道"]
    B --> C3["计算结果泳道"]
    B --> C4["输出成果泳道"]
    C4 --> D1["细项卡片 A"]
    C4 --> D2["细项卡片 B"]
```

优点：
- 状态对比直观
- 看“哪里堵住了”很快
- 视觉上比纯列表好很多

缺点：
- 文件和摘要一多，卡片会拥挤
- 对查看具体文件不如双栏结构舒服

**方案 3**
左树右详情

这是最稳的工程型方案，适合文件、细项、版本持续变多的情况。

展示方式：
- 左边只保留紧凑树：`阶段 -> 分类 -> 细项`
- 右边详情区展示当前节点的：摘要、最终版文件、责任人、更新时间、状态
- 左边树只做导航，不承担内容展示

施工阶段：
- 左侧树在 `施工图阶段` 下增加二级标签节点
- 例如：`施工图阶段 -> 图纸成果 / 设计变更 / 问题闭环 / 归档版本`

示意图：
```mermaid
flowchart LR
    A["左侧树\n阶段 -> 分类 -> 细项"] --> B["右侧详情面板"]
    B --> C1["摘要"]
    B --> C2["最终版文件"]
    B --> C3["状态"]
    B --> C4["更新时间"]
```

优点：
- 最稳定
- 最适合长期演进
- 文件多时不乱

缺点：
- 第一眼没有方案 1 那么“好看”
- 主页感会更偏系统工具，不像项目总览

**我的建议**
如果你现在要的是“既能看整体，又别太丑，还要方便继续做”，我建议：

1. 子项主页用 `方案 1`
2. 施工阶段加二级标签
3. 右侧详情里只展示最终版文件和摘要
4. 如果后面数据量再上来，再演进到 `方案 3`

也就是说，不建议直接做成纯树，不建议继续堆竖向列表，也不建议现在就做很重的可视化图。

**施工阶段二级标签建议口径**
我建议先固定成这 4 个：

1. `图纸成果`
2. `设计变更`
3. `问题闭环`
4. `归档版本`

这套口径比继续沿用通用分类更适合施工阶段。

你如果要我继续，我建议你直接回复一个编号：

1. 选方案 1
2. 选方案 2
3. 选方案 3
4. 方案 1 为主，但我先把施工阶段二级标签口径再细化一版
