# 暖通设计管家｜需求说明书（V1）

- 文档版本：`V1.1`
- 文档状态：`Draft`（已合并本轮修改意见）
- 更新日期：`2026-02-23`
- 适用范围：本地/局域网内的暖通设计过程记录与追溯

## 1. 文档目标

本版聚焦把系统从“可记录”升级为“可持续使用”，确保：

1. 设计人员日常可用
2. 项目负责人可看进度和风险
3. 资料可追溯、可归档、可导出

## 2. 当前问题与已纳入需求

### 2.1 账号与登录

现状问题：
1. 注册逻辑和内部使用场景不一致
2. 用户不理解 JWT 概念

V1.1 需求：
1. 前台不提供自由注册入口（默认关闭）
2. 账号由管理员统一创建，用户名与内网 IM 用户名一致
3. 初始密码统一 `123456`
4. 用户首次登录强制修改密码
5. JWT 仅作为系统内部鉴权机制，不在业务文档中作为用户操作概念

### 2.2 项目与子项主数据

现状问题：
1. 项目/子项允许普通用户创建，容易造成命名混乱
2. 项目名与编号、子项名与编码缺少主数据约束

V1.1 需求：
1. 项目与子项仅允许管理员在后台维护
2. 设计用户仅能“选择”已建主数据，不可新增主数据
3. 支持输入前缀快速匹配项目与子项
4. 项目名称必须绑定项目编号，子项名称必须绑定子项编码

### 2.3 模板与任务体系

现状问题：
1. 当前模板分类过于粗糙，三阶段差异不足

V1.1 需求：
1. 模板按阶段差异化定义，不再使用单一分类思路
2. 方案阶段重点：专业内部设计 + 少量跨专业接口
3. 初步设计阶段重点：专业内部设计 + 多专业接口 + 风险控制
4. 施工图阶段重点：在初设基础上新增设备审查与闭合项
5. 模板继续支持版本化与项目级复用

### 2.4 文件上传与导航体验

现状问题：
1. 存在“文件上传失败”反馈
2. 进入分类后退出不直观
3. 缺少项目整体最终成果打包下载

V1.1 需求：
1. 文件上传失败必须给出明确错误原因（鉴权、大小、网络、后端异常）
2. 文件上传失败支持一键重试
3. 分类导航使用树形结构，支持展开/收起与快速返回总览
4. 新增“整项目最终版一键下载”（按阶段或全项目导出 zip）

### 2.5 知识库维护

现状问题：
1. 规范条文和常见问题为静态数据，不利于扩充

V1.1 需求：
1. 提供后台维护入口，支持新增/编辑/删除规范条文与常见问题
2. 支持按项目类型、阶段过滤推荐
3. 支持 JSON 导入导出

## 3. V1 范围定义（更新后）

### 3.1 V1 必做（Must Have）

1. 管理员主数据机制（账号/项目/子项后台维护）
2. 三阶段差异化模板体系
3. 任务状态管理（未开始/进行中/阻塞/完成）
4. 文件上传可诊断、可重试、可下载
5. 树形分类导航与总览回退
6. 阶段成果包导出 + 整项目最终版导出
7. 知识库后台维护能力

### 3.2 V1 可选（Should Have）

1. 任务评论
2. 操作日志（谁、何时、做了什么）
3. 历史阶段与当前阶段对比视图

### 3.3 V1 不做（冻结）

1. 自动规范合规判断
2. 图纸/文档内容自动解析
3. AI 自动结论输出
4. 重型审批流与复杂权限矩阵

## 4. 关键业务需求（FR）

### FR-01 账号与权限（轻量）

1. 管理员统一创建账号，默认密码 `123456`
2. 首次登录强制修改密码
3. 设计用户仅使用，不维护主数据

### FR-02 项目主数据

1. 项目主数据：`项目名 + 项目编号`
2. 子项主数据：`子项名 + 子项编码 + 类型`
3. 用户侧支持前缀搜索选择

### FR-03 阶段化模板

1. 模板按 `子项类型 + 设计阶段` 下发
2. 三阶段分类与条目可独立配置
3. 施工图阶段必须含设备审查类条目

### FR-04 文件管理

1. 文件按任务与阶段归档
2. 上传失败需返回可读错误信息
3. 文件检索至少支持：阶段/上传人/标签
4. 支持阶段导出与项目总导出

### FR-05 界面交互

1. 左侧分类导航树支持展开/收起
2. 任一分类可一键返回“项目设计情况”总览
3. 历史阶段只读可见

### FR-06 知识库

1. 条文与常见问题改为可维护数据源
2. 支持后台录入与更新
3. 支持导入导出与检索

## 5. 非功能需求

1. 稳定性
- 上传、删除、下载失败要可重试，不得破坏数据一致性

2. 可维护性
- 前端继续模块化（hook + component）
- 后端继续 FastAPI 单路线

3. 安全性
- 下载必须鉴权
- 文件访问路径必须限制在服务目录内

## 6. 验收标准（V1）

1. 账号与主数据流程跑通（管理员建、设计用户选）
2. 三阶段模板差异可配置并生效
3. 文件上传失败可定位原因，且可重试
4. 支持阶段成果导出与整项目最终版导出
5. 知识库可由后台持续维护

## 7. V2 入口（仅方向）

1. 规则引擎（规范条文关联/提示）
2. 轻量协作扩展（评论、通知、简单校审）
3. 不纳入文档解析能力（保持工具轻量化定位）
