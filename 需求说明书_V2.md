# 暖通设计管家｜需求说明书（V2）

- 文档版本：`V2.0`
- 文档状态：`Draft`
- 更新日期：`2026-03-20`
- 适用范围：本地/局域网内的暖通设计过程记录、模板管理与设计追溯
- 基于文档：`需求说明书_V1.md`、`暖通设计管家｜需求说明书（基础版 v0）`、`PLAN.md`

## 1. V2 升级背景与定位

### 1.1 V1 遗留问题

V1 已落地的页面逻辑与用户核心诉求之间存在偏差：

| 问题 | 说明 |
|------|------|
| **核心诉求被淹没** | 用户的核心需求是"记录设计过程 + 做设计模板"，但当前页面以"任务勾选 + 完成度统计"为主线，设计过程记录沦为二级操作 |
| **三视图定义偏离** | dashboard 偏重任务状态追踪；regulations 与 errors 为辅助知识库；缺少"设计过程"和"模板管理"的一级视图 |
| **模板管理缺失** | 模板仅作为 `constants.ts` 硬编码常量，无独立管理、导入/导出界面 |
| **设计过程记录不明显** | 文件版本管理隐藏在任务条目子级中，缺少"设计过程"的一级全局视角 |

### 1.2 V2 核心定位

> **V2 要让"记录设计过程"和"做设计模板"成为系统的第一公民，而非附属功能。**

V2 在保持 V1 所有已冻结约束（不做自动判断、不解析文件内容、不做审批流程）的前提下，对页面逻辑进行重新组织：

- 设计过程记录 → 从二级操作提升为一级视图
- 模板管理 → 从硬编码常量提升为独立可维护页面
- 图纸提取模板 → 新增能力，支持从已有图纸提取模板

## 3. 新增功能需求

### FR-07 设计过程视图（Design Process View）

#### 3.1 设计过程时间线

**功能描述**：以时间线形式记录子项在当前阶段的所有设计活动，包括：

- 文件上传/版本更新
- 任务状态变更（开始/完成/阻塞）
- 阶段推进
- 人工添加的设计备注/说明

**展示方式**：

- 时间线按时间倒序排列（最新在上）
- 每条记录包含：时间、操作人、操作类型、操作内容、关联条目
- 支持按操作类型过滤（文件类/状态类/备注类）
- 支持按时间范围筛选

**交互行为**：

- 设计人员可随时手动添加"设计备注"（自由文本，不超过 500 字）
- 设计备注支持关联到某个分类或任务条目（可选）
- 时间线条目可展开查看详情
- 点击文件相关条目可直接跳转到对应的任务条目

#### 3.2 设计概览面板

**功能描述**：在设计过程视图顶部，展示当前子项的设计全貌：

- 当前阶段 + 进入当前阶段的日期
- 本阶段累计设计活动次数
- 本阶段已上传文件数
- 各分类完成度快速概览（沿用 V1 口径）

> 该面板不替代子项主页的完成度详情，仅作为设计过程视角的快速感知。

#### 3.3 设计过程导出

- 支持将设计过程时间线导出为 Markdown 或 PDF 格式
- 导出内容包含：项目/子项信息、阶段、时间线记录

---

### FR-08 模板管理视图（Template Management View）

#### 3.4 模板列表与管理

**功能描述**：提供独立的模板管理界面，展示所有可用模板：

- **系统内置模板**：按 `子项类型 + 设计阶段` 组织（沿用 V1 `TEMPLATE_CATEGORIES` 结构）
- **用户自定义模板**：设计人员基于内置模板修改后保存的模板

**模板操作**：

| 操作 | 说明 |
|------|------|
| 查看 | 展开模板，查看所有一级/二级分类及条目 |
| 编辑 | 修改模板名称、增删分类、增删条目 |
| 复制 | 从现有模板复制为新模板 |
| 删除 | 删除用户自定义模板（内置模板不可删除） |
| 导出 | 导出模板为 JSON 文件 |
| 导入 | 从 JSON 文件导入模板 |

#### 3.5 从图纸提取模板

**功能描述**：用户上传多个已有工程图纸（PDF/图片），系统提取其中的分类与条目结构，生成新模板。

**流程**：

1. 用户点击"从图纸提取模板"
2. 选择并上传一个或多个图纸文件
3. 系统解析图纸中的文件分类信息（V2 阶段为人工标注辅助，不做自动 OCR 解析）
4. 用户在系统中手动标注/填写从图纸中识别到的分类和条目
5. 系统生成新模板，用户可预览并保存

> [!IMPORTANT]
> V2 中"从图纸提取模板"**不做自动解析**，仅提供结构化的人工录入界面来辅助提取过程。自动解析为 V3+ 能力方向。

#### 3.6 模板应用与关联

- 新建子项时，可直接选择用户自定义模板（在内置模板列表中追加显示）
- 模板一旦应用到子项，后续模板修改不影响已创建的子项（沿用 V1 规则）

## 4. 页面逻辑调整需求

### FR-09 子项主页逻辑调整

V2 对现有子项主页（dashboard）进行以下微调：

1. **增加"设计过程入口"按钮**：在极简执行视图区域增加跳转到设计过程视图的快捷入口
2. **增加"最近设计活动"摘要**：在概览区域展示最近 3 条设计活动记录（摘要级别）
3. **保持所有 V1 功能不变**：任务勾选、完成度、阶段管理、文件上传、导出等全部保留

### FR-10 侧边栏导航调整

1. 功能菜单从 3 项扩展为 5 项（见 2.2 节）
2. `ViewState` 类型扩展为：`'process' | 'dashboard' | 'templates' | 'regulations' | 'errors'`
3. 默认选中视图从 `dashboard` 改为 `process`

## 5. 数据结构扩展

### 5.1 设计活动记录（DesignActivity）

```typescript
interface DesignActivity {
  id: string;
  subProjectId: string;
  stage: DesignStage;
  type: 'FILE_UPLOAD' | 'STATUS_CHANGE' | 'STAGE_ADVANCE' | 'NOTE' | 'TASK_ADD' | 'TASK_DELETE';
  actor: string;
  createdAt: string;
  content: string;              // 活动描述
  relatedTaskId?: string;       // 关联的任务条目 ID
  relatedCategoryId?: string;   // 关联的分类 ID
  metadata?: Record<string, any>; // 扩展信息
}
```

### 5.2 用户模板（UserTemplate）

```typescript
interface UserTemplate {
  id: string;
  name: string;
  description?: string;
  baseType: ProjectType;       // 基于的项目类型
  baseStage: DesignStage;      // 基于的设计阶段
  categories: TemplateCategory[];
  createdBy: string;
  createdAt: string;
  updatedAt: string;
  isBuiltIn: boolean;          // 是否为系统内置
}
```

### 5.3 后端 API 扩展

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/activities?subProjectId=&stage=` | 获取设计活动列表 |
| POST | `/api/activities` | 添加设计备注 |
| GET | `/api/templates` | 获取所有模板列表 |
| POST | `/api/templates` | 创建新模板 |
| PUT | `/api/templates/:id` | 更新模板 |
| DELETE | `/api/templates/:id` | 删除模板 |
| POST | `/api/templates/import` | 导入模板（JSON） |
| GET | `/api/templates/:id/export` | 导出模板（JSON） |

## 6. V2 范围定义

### 6.1 V2 必做（Must Have）

1. 设计过程时间线视图（FR-07）
2. 设计概览面板（FR-07.2）
3. 模板管理视图（FR-08，含查看/编辑/复制/删除/导入/导出）
4. 页面视图扩展为 5 视图架构（FR-10）
5. 侧边栏导航调整（FR-10）
6. 设计活动数据结构与 API（5.1, 5.3）
7. 用户模板数据结构与 API（5.2, 5.3）

### 6.2 V2 可选（Should Have）

1. 设计过程导出为 Markdown/PDF（FR-07.3）
2. 从图纸提取模板的人工辅助界面（FR-08.5）
3. 子项主页增加设计过程入口与摘要（FR-09）

### 6.3 V2 不做（冻结）

1. 自动 OCR 解析图纸内容（V3+）
2. AI 辅助模板生成（V3+）
3. 多人协作与审批流程（V3+）
4. 设计过程的自动分析与风险预警（V3+）

## 7. 与 V1 的关系

### 7.1 保持不变

- V1 的所有已冻结约束继续有效
- V1 的所有已实现功能（任务勾选、完成度、阶段管理、文件管理、知识库）保持不变
- V1 的数据结构不做破坏性修改

### 7.2 扩展变更

- `ViewState` 类型扩展（增加 `process` 和 `templates`）
- 功能菜单从 3 项扩展为 5 项
- 默认首页从 `dashboard` 改为 `process`
- 新增 `DesignActivity` 和 `UserTemplate` 数据结构
- 后端新增活动和模板相关 API

## 8. 验收标准（V2）

1. 进入系统后默认展示"设计过程"视图，时间线可正常显示设计活动记录
2. 可以在设计过程视图中手动添加设计备注
3. 模板管理视图可查看内置模板，可创建/编辑/删除用户自定义模板
